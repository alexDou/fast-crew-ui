## Codebase Patterns
- Path aliases in tsconfig.json must be mirrored in vitest.config.ts resolve.alias for tests to work
- Use `yarn test:run` for CI (single run), `yarn test` for watch mode
- Playwright browsers installed via `npx playwright install`; Kali Linux needs `sudo npx playwright install-deps` for system libs
- Biome is the formatter (not Prettier). Use `biome format --write` and `biome check --write --unsafe` for formatting + import sorting + class sorting
- API constants: `API_ENDPOINTS` for fast-api paths, `BFF_ENDPOINTS` for Next.js proxy routes, `ERROR_MESSAGES` for error strings â€” all in `@/constants/api`
- ky `prefixUrl` ignores URLs starting with `/` â€” use a `normalizeUrl()` helper to strip leading slashes for backward compatibility with existing path constants
- CSS files with Tailwind directives require `css.parser.tailwindDirectives: true` in biome.json
- ESLint flat config: eslint-config-next already registers jsx-a11y plugin â€” don't redefine it in custom config objects
- For same-origin BFF routes (paths starting with `/`), use plain `ky` import â€” not the `api` instance from `@/lib/api.ts` which has `prefixUrl` for the external API
- Use `vi.hoisted()` to define mock variables referenced inside `vi.mock()` factories â€” vi.mock is hoisted above imports
- Mock `@/env` in any test importing modules that depend on env validation: `vi.mock("@/env", () => ({ env: { NEXT_PUBLIC_API_URL: "https://api.example.com" } }))`
- @testing-library/dom must be installed as devDep (peer of @testing-library/react)
- For hook tests with TanStack Query: use `retryDelay: 0` and `gcTime: 0` in test QueryClient; wrapper via `React.createElement(QueryClientProvider, ...)`
- Mock next-intl: `vi.mock("next-intl", () => ({ useTranslations: () => (key: string) => key }))`
- vitest.config.ts alias resolution uses array format `[{ find, replacement }]` with `@` catch-all LAST â€” object format lets `@` intercept `@/ui`, `@/hooks`, etc.
- For server component pages: use `getTranslations()` / `getFormatter()` from `next-intl/server`
- Mock `@/ui` in page tests to avoid resolving the full shadcn barrel dependency chain
